### Issues Found & Fixed

#### [HIGH] Race condition on model update during active generation
**Source:** Gemini + Codex (cross-referenced)
**File:** `kling_gui/main_window.py:857-880`
**Problem:** `update_model()` was called immediately on any config change, even while processing was active. This could cause a mid-run model switch.
**Fix Applied:**
- Check `queue_manager.is_running` before updating generator
- If processing active, log warning and skip update (config still saved)
- Sync generator with latest config when queue becomes empty (`_on_item_complete`)

#### [HIGH] video_duration never applied to API requests
**Source:** Codex
**File:** `kling_generator_falai.py:221-227`
**Problem:** Duration was hardcoded as `"10"` in the API payload. The `video_duration` config setting was logged but never used.
**Fix Applied:**
- Added `duration` parameter to `create_kling_generation()`
- Updated payload to use `str(duration)` instead of hardcoded `"10"`
- Updated `queue_manager.py` to pass `video_duration` from config

#### [MEDIUM] Model updates unsynchronized with worker thread
**Source:** Gemini
**File:** `kling_generator_falai.py:54-63`
**Problem:** `update_model()` mutates attributes without locking, could cause inconsistent state.
**Fix Applied:** Resolved by gating updates behind `is_running` check in main_window.py

#### [MEDIUM] Other settings may have same bug pattern
**Source:** Gemini
**Analyzed:** `verbose_logging`, `video_duration`, prompt slots
**Status:** `video_duration` fixed. `verbose_logging` has no UI to change it (CLI only). Prompt slots are read dynamically per-item (no issue).

#### [LOW] verbose_logging changes don't propagate after init
**Source:** Codex
**File:** `kling_gui/main_window.py:607-612`
**Problem:** `verbose` is set once at generator construction.
**Status:** NOT FIXED (low priority - no UI to change this setting in GUI)

#### [LOW] Broad try-except during Tkinter cleanup
**Source:** Gemini
**File:** `kling_gui/main_window.py:1026`
**Problem:** Broad exception handling may mask cleanup issues.
**Status:** NOT FIXED (acceptable for shutdown robustness)
